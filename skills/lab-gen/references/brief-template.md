# Experiment brief — the step-1 output template

Studying the material always ends by filling in this brief. It bounds the study
step, it is the faithfulness contract for everything downstream — **every
control, param, stat, and sim behaviour must trace to a line in the brief** —
and it becomes most of your final report to the user.

If the user supplies a written brief in (roughly) this shape instead of images,
accept it as-is and skip image study entirely.

## Template

```
EXPERIMENT: <one line — what is being demonstrated or measured>
SOURCE: <textbook/page/figure ids, or URL/site name for web material>

APPARATUS: <each physical item the figures/procedure show>

METHODS / VARIANTS: <the distinct ways the material offers to run the
  experiment — these become the method dropdown options, one each, no inventions.
  A single method is fine.>

FIXED QUANTITIES: <values the material states as given (a frequency, a recipe
  amount, a concentration) — stay fixed in the lab unless the material itself
  varies them>

ADJUSTABLE PARAMETERS: <quantities a student could plausibly set per run,
  with the material's own value as the default and a sensible range around it>

MEASURED / DISPLAYED QUANTITIES: <what the apparatus reads out or the student
  measures — these become stat cards; note measured-vs-true discrepancies the
  method implies>

MEASUREMENT EVENTS: <the moments that trigger or freeze a reading (beam broken,
  dot printed, reaction complete)>

EXCLUDED: <pages/sections given but not used (outlines, unrelated topics,
  sections referenced but not provided) and one clause on why>
```

## Rules

- Fill every field; write `none` rather than leaving one blank.
- METHODS and ADJUSTABLE PARAMETERS may only contain what the material shows —
  if you are tempted to add a variant "for pedagogy", it goes in EXCLUDED with
  the reason, not in the lab (see SKILL.md Rules).
- Multi-page material: read ALL pages in one pass before filling anything in.
- Keep the brief under ~30 lines; it is a contract, not a summary essay.
